Grid layouts

The default Bootstrap grid is used. Any set of div(ision)s must be placed within a 'row'-classed div.

Content containers

A content container is any div placed within another grid div (featuring only classes beginning "col-") on a page. The class 'box' should be assigned to all content containers unless they are required to be transparent. CSS classes can be added to the content container to over-ride the default 'box' attributes and/or to specify width, margin, padding, color and borders for the div. The div this text is contained within includes a background color of black (rgba 0,0,0) with 25% opacity, padding of 15 pixels (deemed 'lg' as in large) on all sides and a border radius of 5 pixels on all corners.

A box within a box

This is a content container with 'medium' all-round padding (10 pixels), a background color of black (rgba 0,0,0) with 50% opacity and a 5 pixel border radius. The padding, color and border radius are all set with individual classes.

Headings within content containers

This is the h3 heading, shaded, within a container

But normally H3 headings are only used above content containers due to the padding and margin applied to the H3 creating a noticeable gap between it and the top of the container. The H3 margin can be over-ruled by applying the 'mar-all-none' class to the heading. In doing so an additional top margin of 15 pixels will be automatically applied to any element subsequent to the heading e.g. a paragraph of text.

This is the h3 heading, shaded and with 'mar-all-none' applied

And this subsequent paragraph duly has a top margin of 15 pixels applied via the CSS file rather than through a class reference in the HTML.

This is the h4 heading

h4 headings can appear above or within content containers provided H3 headings also appear before them.

Horizontal ruler

The horizontal rule can be used to divide sections of content visually. Place it within a full-width column div (col-xs-12) before placing it between different content elements (not including headings and paragraphs).

    This UL format is useful as it allows irregular heights among list items without the 'nesting' effect of the Bootstrap-based column containment.     On every categorised page

    Body

    Unless styled otherwise text is displayed in #ccc (Gray 80) in the webfont Roboto Condensed. Unless styled otherwise text links are coloured #6cc are underlined when hovered over. A class for a background colour is applied to the page according to which category the page falls under. A background image is applied to the body positioned absolutely from the top right.

    Header

    Aside from the homepage the header on every page contains only a link back to the homepage. This link contains a heavy round-tipped rightwards arrow (Unicode character 'U+279C') rotated with a CSS class by 135 degrees anti-clockwise within a container with a fully-rounded border. This is followed by the title 'At A Glance' displayed white with an opacity of .75 and in the webfont 'Tex Gyre Heros' provided from the toshlist.org webfonts directory.

    Footer

    The footer on every page provides a link to the site creators, the Toshlist.org homepage and a link back to the top of the page (again using the heavy round-tipped rightwards arrow (Unicode character 'U+279C') but rotated with a CSS class by 90 degrees counter-clockwise. The footer has a minimum height of 200 pixels and is coloured black with an opacity of .85.

    The primary container

    This appears on every page in a container below the header and contains the H1 and H2 headings
